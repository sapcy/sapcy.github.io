## 1. Multiple DNS server forwarding
- 특정 도메인이 한대의 DNS 서버에만 쿼리가 가능한 경우에 간헐적으로 lookup이 실패하는 이슈
	- 해결 1. policy sequential 사용
	  .:53 {
	    forward . 8.8.8.8 9.9.9.9 {
	        policy sequential; # Use sequential policy
	    }
	    # Other configurations for your CoreDNS setup
	  }
	  
	- 해결 2. 도메인 별로 zone 설정

- CoreDNS는 일반적으로 라운드 로빈 방식으로 DNS 쿼리를 여러 DNS 서버에 전달합니다. 그러나 구성된 DNS 서버의 하위 집합만 쿼리하고 초기 서버가 실패한 경우 후속 서버는 쿼리하지 않는 설명된 동작은 다양한 요인의 영향을 받을 수 있습니다.

1. **DNS 서버 가용성:** CoreDNS가 DNS 서버를 사용할 수 없거나 특정 시간 초과 기간 내에 응답하지 않는다고 판단하면 해당 서버를 건너뛰고 다음 서버로 이동할 수 있습니다. 이 동작은 응답하지 않는 서버를 무기한 기다리지 않음으로써 쿼리 응답 시간을 향상시키기 위한 것입니다.
2. **DNS 서버 상태:** CoreDNS는 DNS 서버의 상태 확인 또는 모니터링을 사용하여 가용성을 확인할 수 있습니다. 서버가 비정상으로 표시되면 다시 정상으로 간주될 때까지 라운드 로빈 순환에 포함되지 않을 수 있습니다.
3. **구성 오류:** CoreDNS 설정에 IP 주소나 포트 번호의 오타와 같은 구성 오류가 없는지 확인하세요. 구성에 오류가 있으면 예기치 않은 동작이 발생할 수 있습니다.
4. **DNS 캐싱:** 많은 DNS 확인자와 마찬가지로 CoreDNS는 특정 기간 동안 DNS 응답을 캐시할 수 있습니다. 특정 도메인에 대한 쿼리가 이미 응답되어 캐시된 경우 CoreDNS는 캐시된 응답이 여전히 유효하다고 가정하여 동일한 쿼리를 다른 서버로 보내지 않을 수 있습니다.
5. **DNS 확인 정책:** 쿼리 처리 방법에 영향을 줄 수 있는 특정 DNS 확인 정책이나 플러그인이 CoreDNS 구성에 있는지 확인하세요. 'rewrite', 'route' 또는 사용자 정의 미들웨어와 같은 일부 플러그인은 DNS 쿼리 동작을 변경할 수 있습니다.
6. **로깅 및 디버깅:** CoreDNS에서 디버깅 또는 자세한 로깅을 활성화하면 쿼리 처리 방법과 사용 중인 DNS 서버에 대한 자세한 정보를 얻을 수 있습니다. 이는 문제를 진단하는 데 도움이 될 수 있습니다.

CoreDNS가 쿼리할 수 있는 DNS 서버 수에는 엄격한 제한이 없습니다. 일반적으로 구성된 모든 DNS 서버가 모두 정상적이고 응답성이 있는 경우 라운드 로빈 방식으로 쿼리해야 합니다. 서버의 하위 집합만 쿼리되는 문제가 발생하는 경우 위에서 언급한 요소를 조사하여 근본 원인을 파악하고 이에 따라 구성을 조정하는 것이 중요합니다.
