### **연결 관련 커널 옵션 정리**
- 서비스 간 통신이 빈번해지면서 사용 가능한 포트가 모두 소진되어 새로운 연결 요청 처리가 불가능한 상황
  => 시스템 자원이 여유가 있다면 TIME_WAIT 상태의 소켓을 재사용 할 수 있는 *net.ipv4.tcp_tw_reuse*를 적용하여 서비스 처리량 높임.
- 대규모 프로모션과 같은 이벤트 기간 때 짧은 시간에 과도한 트래픽이 몰리는 경우 요청 누락 문제 발생할 수 있음
  => 수신 버퍼 크기를 늘리는 커널 옵션인 net.core.rmem_max를 늘리면 요청 누락을 방지할 수 있음.

### **컨테이너 기술과 OS 커널**
- 컨테이너는 네임스페이스를 활용하여 호스트 운영 체제와 격리된 환경을 제공하며, 동일한 커널을 공유하면서도 프로세스, 네트워크, 파일 시스템 등의 리소스를 독립적으로 관리함. 
  => 애플리케이션 배포 및 관리의 효율성과 보안성을 높임.
- 다음은 컨테이너 런타임 엔진인 Containerd를 사용해 컨테이너 프로세스를 동작 중인 노드의 구조도.
  ![](http://s-core.co.kr/wp-content/uploads/2024/06/%EA%B7%B8%EB%A6%BC2-container-kernel-space.png)
- 컨테이너는 호스트 운영체제의 커널을 공유하며, 커널이 제공하는 기능과 자원 격리 메커니즘 기반으로 동작.
- 커널은 컨테이너에 필수적인 네임스페이스, cgroup, 커널 기능과 같은 핵심 기술을 제공.
- *네임스페이스*를 통해 프로세스, 네트워크, 마운트 등의 리소스가 격리됨.
- *cgroups*를 통해 CPU, Memory, Disk I/O 등의 리소스 사용이 제한됨. 
- 커널의 기능들이 컨테이너의 보안과 효율성을 뒷받침.
- 커널은 컨테이너 기술의 근간이 되며, 컨테이너 런타임과 오케스트레이터는 커널이 제공하는 기능을 활용하여 컨테이너를 생성, 관리, 실행.

### **커널 옵션 튜닝의 필요성**
적절한 OS 커널 옵션 설정은 다음과 같은 이점을 제공한다.
* *성능 향상:* 컨테이너 네트워킹 속도 향상, 메모리 사용량 최적화, 컨테이너 스케줄링 효율 개선 등을 통해 성능을 향상 시킬 수 있다.
* *비용 절감:* 리소스 사용 효율성을 높여 클라우드 비용을 절감할 수 있다.
* *안정성 강화:* 시스템 오류 및 컨테이너 실패 가능성을 줄여 안정성을 높일 수 있다.
