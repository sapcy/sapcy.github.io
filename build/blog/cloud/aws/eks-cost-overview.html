요금 분석​

1. EC2​

VM 유형별 요금​

  • t2.small: 만들고 30분뒤 삭제한 노드 VM
    -> $0.023 / hour × 0.521 hrs ≈ $0.01

  • t2.medium: 8시간 정도 사용한 노드 VM
    -> $0.0464 / hour × 8.399 hrs ≈ $0.39

  • c6g.large: EKS Auto Mode로 인해 자동으로 생성된 관리 노드 (2대)
    -> $0.068 / hour × 17.678 hrs ≈ $1.20

c6a.large, c6g.large 비용이 왜 포함되었을까 ?​

EKS 자동 모드는 클러스터 생성시 AMI(aws-k8s-1.xx-standard)를 사용하여 CPU 전용으로 metrics-server 를 실행하는 데 사용되는 c6x 유형으로 두 개의 인스턴스를 실행한다.

*관리형 인스턴스의 유형은 사용자가 임의로 설정이 불가능하다.

관리형 인스턴스와 표준 EC2 인스턴스의 차이는 아래 비교표를 통해 알 수 있다.

표준 EC2 인스턴스
EKS 자동 모드 관리형 인스턴스
패치 및 업데이트는 사용자의 책임AWS에서 자동으로 패치 및 업데이트를 적용
EKS는 인스턴스에 설치된 소프트웨어에 대해 책임을 지지 않음.EKS는 컨테이너 런타임, 운영 체제, kubelet 같은 인스턴스에 설치된 특정 소프트웨어들을 책임진다.
EC2 API를 사용하여 EC2 인스턴스 삭제 가능.EKS는 계정에 배포된 인스턴스 수를 결정.
워크로드를 삭제하면 EKS는 계정의 인스턴스 수를 줄임.
SSH를 사용하여 EC2 인스턴스에 접속 가능.관리형 인스턴스에 파드와 컨테이너를 배포 가능.
운영 체제와 이미지(AMI)를 사용자가 결정.AWS가 운영 체제와 이미지를 결정.
Windows 또는 Ubuntu 기능을 사용하는 워크로드를 배포 가능.리눅스 기반 컨테이너를 배포할 수 있지만 특정 운영체제에 대한 종속성은 없음.
실행할 인스턴스 유형 및 제품군을 사용자가 결정.AWS가 어떤 인스턴스 유형과 패밀리를 선택하여 시작할지 결정.
노드 풀을 사용하면 EKS 자동 모드가 선택하는 인스턴스 유형을 제한할 수 있음.

2. EBS​

스토리지 유형별 요금​

  • gp3 20GB
    -> $0.08 / GB-month × 2.525 GB-Mo ≈ $0.20

3. EKS Control Plane​

클러스터 생성 순간부터 Control plane에 대한 과금 발생

-> $0.10 / hour × 9.284 hrs ≈ $0.93

클러스터 유지 비용이라 봐도 무방

4. EKS Auto mode​

EKS Auto mode 사용시 EC2 및 Control plane 비용과 별개로 Auto mode 관리 비용이 따로 발생한다.

  • c6g.large
    -> $0.00816 / hour × 17.678 hrs ≈ $0.14
  • c6a.large
    -> $0.00918 / hour × 2.518 hrs ≈ $0.02

5. VPC​

EKS 생성시 선택한 Subnet이 전부 Auto-assign public ip 설정이 되어있었으므로 EC2 인스턴스들이 전부 public IP가 할당되었다.

그러므로 비용 계산은 [EC2 전체 사용량의 합계 X $0.005]가 된다.

-> $0.005 × 29.221 hrs = $0.146 ≈ $0.15
